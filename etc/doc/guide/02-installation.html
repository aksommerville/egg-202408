Prerequisites

Clone wasm-micro-runtime: https://github.com/bytecodealliance/wasm-micro-runtime , and build with cmake.

You'll need a WebAssembly compiler. I recommend clang.

Strongly recommend the WebAssembly Binary Tools. Clone from https://github.com/WebAssembly/wabt or check your package manager.

First Build

Clone Egg: https://github.com/aksommerville/egg

make, and it will generate a config file and abort. That happens only the very first time.

Edit etc/config.mk with paths to your wamr and wabt installations, and the targets you want to build for.

make again, and it should build the whole project. A full build from scratch takes about 8 seconds on my 2021 PC running Linux.

Anything fails during make, check your settings in config.mk, maybe you can fix it. Anything you can't fix, please open an issue in Egg's Github repo.

Now that it's built, try running the included example ROM: `make run`

Build the Eggsamples

I provide a set of example games, many with extensive client-side utilities that you'll probably find helpful. You can use these as starting points for your game, or cannibalize them bit-by-bit, or just read as a reference, whatever suits you.

Clone: https://github.com/aksommerville/eggsamples
Best to clone it right next to egg, it will look there by default.

`make` from the root of that repo will build all the games.

Then you can `make run-NAME` to try them. The current list:

  • lightson: Tests platform features.
  • shmup: Tiny space shooter.
  • hardboiled: Point-and-click detective story.
  • leggend: Zelda style adventure.

Each Eggsample project has pretty much the same Makefile, and you'd be wise to copy it verbatim for new projects.
